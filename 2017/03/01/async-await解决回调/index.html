async/await解决回调

发表于 2017-03-01 |

异步方案有很多种，目前比较好的就是es6中的promise,典型的例子就是fecthAPI
而es7中的async/await则更加的好。

第一个例子

 var sleep = function(time) {
    return new Promise(function(resolve,reject) {
            setTimeout(function() {
                    resolve()
                },time)
        })
}
var start = async function() {
    //在这里使用就像同步代码那样直观
    console.log('start')
    awit.sleep(3000)
    console.log('end')
}

基本规则

async表示这是一个async函数，await只能用在这个函数里面。
await表示在这里等待promise返回结果了，再继续执行
await后面跟着的应该是一个promise对象(当然，其他返回值也没关系，只是会立即执行，不过那样就没有意义了)

获得返回值

await等待的虽然是promise对象，但不必写.then(…),直接可以得到返回值
var start = async function() {
let result = await sleep(3000);
console.log(result);//收到”ok”
}

捕捉错误

既然.then(..)不用写了，那么.catch(…)也不用写，可以直接用标准的try catch 语法捕捉错误。
var sleep = function() {
return new Promise(function(resolve,reject) {
setTimeout(function() {
//模拟出错
reject(‘error’)
},error)
})
}
var start = async function() {
try {
console.log(‘start’)
await sleep(3000);//这里得到一个返回错误
//以下代码不会被执行
console.log(;end)
}
catch(err) {
console.log(err)//这里捕捉到了错误
}
}
