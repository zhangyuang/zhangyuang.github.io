学习笔记之'use strict'

发表于 2016-11-20 |

概念

如果给javascript代码标志为“严格模式”，则其中运行的所有代码都必然是严格模式下的。
其一：如果在语法检测时发现语法问题，则整个代码块实效，并导致一个语法异常。
其二：如果在运行期出现了违反严格模式的代码，则抛出执行异常。
注：经过测试ie6,7,8,9均不支持严格模式。
使用方法，添加“use strict”
在如下位置加入可以开启相应代码块中的严格模式：
1、必须在全局代码的开始处加入。
2、在eval代码开始处加入。
3、在函数声明代码开始处加入。
4、在 new function（）所传的body参数块开始加入。

学习笔记之css reset

发表于 2016-11-19 |

含义

流行的浏览器有一些以自己的方式去理解css规范，这样会导致有的浏览器对css的解释与设计师的css定义初衷冲突，使得网页的样子在某些浏览器下能正确显示，但有些浏览器缺并没有按照设计师想要的样子显示出来，这就导致浏览器的兼容性问题，更糟的是，有些浏览器完全无视css的一些声明和属性。
正因为上述冲突和问题依然存在于这个“不完美的世界”，所以一些设计师想到了一种避免浏览器兼容性问题的方法，那就是 Css Reset。可以叫做css重设也可以叫做css抚慰，默认css，css重置等。所以我们要先定义好一些css样式，让所有浏览器按照同样的规则解释css就可以避免发生这种问题。
